## 项目介绍

### 1. 大数据资产管理平台项目介绍：以 AI 一键建标为核心的标签资产生产平台

面向数据治理场景的数据资产与标签管理平台。

平台最核心的能力是 AI 一键建标。业务人员输入自然语言场景后，系统会自动完成意图识别、资源召回、标签方案生成和规则落库；底层再通过 DAG 编排、计算任务执行和 Elasticsearch 索引同步，把建标结果真正沉淀成可检索、可复用的标签资产。

项目后端主要划分为：

- 标签核心服务：资源目录、规则落库、任务编排、标签资产和查询。
- 计算服务：Hive/Spark/Flink 作业提交和任务状态回传。
- 大模型服务：AI 一键建标、RAG 召回、资源推荐和自然语言转 SQL。

我参与的内容主要包括：

- 标签核心服务和标签资产相关功能建设。
- 规则 DAG 解析、Hive SQL 生成和标签任务编排。
- 任务状态跟踪、失败重跑、结果追溯和防重复提交。
- Hive 标签结果同步 Elasticsearch。
- 索引重建、增量追平和 Alias 切换方案。
- AI 一键建标、自然语言意图识别、资源召回和建标结果落库能力设计。

核心技术：

`Java`、`Spring Boot`、`MyBatis`、`MySQL`、`Redis`、`Hive`、`Flink`、`YARN`、`Kafka`、`Elasticsearch`、`Milvus`

### 2. 营销短链与投放归因平台

面向短信、私域、渠道、物料和 App 拉新等场景的营销投放平台。

平台通过短链承接营销流量，记录用户的访问渠道，并将点击、注册、下单和支付事件关联起来，用于分析不同渠道的转化效果和投放 ROI。

系统主要包含两条链路：

- 在线跳转链路：短码解析、缓存查询、访问校验和 302 跳转。
- 数据归因链路：点击采集、业务事件回传、事件幂等和渠道归因。

我参与的内容主要包括：

- 点击、落地、注册、下单和支付等标准事件模型设计。
- 点击事件采集和业务转化事件回传接口。
- `eventId`、`traceId`、活动、渠道、批次和物料等归因字段设计。
- 接口重试和 Kafka 重投场景下的事件幂等处理。
- 点击到注册、下单和支付的归因链路设计。
- 向下游计算服务提供标准事件数据和归因上下文。
- 参与短链缓存、异常访问识别和稳定性方案设计。

核心技术：

`Java`、`Spring Boot`、`MyBatis`、`MySQL`、`Redis`、`Kafka`、`ClickHouse`、`Flink`、`Nginx`
